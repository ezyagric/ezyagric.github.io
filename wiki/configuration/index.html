Basic Configuration

Most of the app configurations are managed remotely via remote configs.

Manually testing a configuration

To manually test a configuration, For example a gateway link.

Traverse to the RemoteConfigUtils.java and replace it's configuration source with your manual version.

// RemoteConfigUtils.java
...
    public String couchBaseUrl() {
        return prefManager.getRemoteConfig("COUCHBASE_URL", "http://XX.XXX.XXX.XXX:XXXX/ezyagric/");
    }
...

Replace with

// RemoteConfigUtils.java
...
    public String couchBaseUrl() {
        return "YOUR_CONFIGURATION";
    }
...

Note: Don't forget to restore the existing configuration before sending a pull request to main branch.

Other configurations

There are other configurations that do not affect the day to day function of the app but provide for clean code and re-usability.

These can be found in the CONFIG.java file that mostly contains video configurations and AppConstants.java that provides preference and other configs.

Note: These configurations should be merged to provide for cleaner code architecture.
